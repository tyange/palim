# 원고지 편집 vs 일반 텍스트 에디팅 — 차이점 분석

> 첨부 이미지(원고지 사용법 도해)를 분석해, 실제 원고지처럼 동작하는 에디터를
> 만들 때 일반 텍스트 에디터와 **무엇이 근본적으로 다른지** 정리한 문서.
> 구현 전 설계 기준으로 삼는다.

---

## 0. 한 줄 요약

일반 에디터는 **"문자열이 흐른다"** (글자 → 가변 폭 → 자동 줄바꿈).
원고지는 **"고정된 칸에 글자를 배치한다"** (칸 = 의미 단위, 글자 종류마다 칸 점유량이 다르고,
줄의 시작/끝과 줄 번호에 따라 들어갈 수 있는 것이 정해져 있다).

핵심 차이는 4가지 축으로 나뉜다.

1. **칸 점유 규칙** — 글자 종류마다 한 칸에 들어가는 개수가 다르다.
2. **구조적 빈 칸 / 줄 규칙** — 들여쓰기·정렬이 "자동 흐름"이 아니라 "줄 번호별 약속"이다.
3. **행두·행말 금칙(禁則)** — 줄 맨 앞/맨 끝에 올 수 없는 글자가 있다.
4. **커서·삭제 모델** — 위치 단위가 "문자 사이"가 아니라 "칸(+칸 내부 위치)"이다.

---

## 1. 칸 점유 규칙 (Cell occupancy) — 가장 큰 차이

일반 에디터에서 모든 글자는 동등하게 "글자 한 개"다. 원고지는 **글자의 종류에 따라
한 칸에 들어가는 개수가 다르다.** 이미지에 명시된 규칙:

| 종류 | 한 칸당 | 비고 |
|------|---------|------|
| 한글·한자·전각 문자 | 1글자 | 기본 단위 |
| 숫자 | **2개** | `25` → 한 칸. `2025` → 두 칸(`20`/`25`) |
| 알파벳 대문자 | 1글자 | `A` 한 칸 |
| 알파벳 소문자 | **2개** | `ab` → 한 칸 |
| 마침표 `.` · 쉼표 `,` | 1칸 | **단, 다음 칸을 띄우지 않음** (일반 텍스트는 보통 뒤에 공백) |
| 물음표 `?` · 느낌표 `!` | 1칸 | |
| 줄임표 `……` | 한 칸에 가운뎃점 **3개**씩 → 보통 **두 칸**(총 6점) | 가운데 정렬 |
| 마침표 + 따옴표 (`."` `.'`) | **한 칸에 같이** | 문장 끝 인용 종료 |

### 일반 에디팅과의 차이
- `"abc123"` 입력 → 일반 에디터는 6칸 폭. 원고지는 `ab`(1칸) + `c`(1칸) + `12`(1칸) + `3`(1칸) = 4칸.
- 즉 **글자 수 ≠ 칸 수**. "텍스트 → 칸 배치"는 단순 1:1 매핑이 아니라
  글자 종류를 분류해 토큰을 묶는 **레이아웃 엔진**이 필요하다.
- 같은 칸에 2글자(숫자·소문자)가 들어가므로, 한 칸 안에서도 **좌/우 두 위치**가 존재한다.

---

## 2. 구조적 빈 칸 / 줄 규칙 (Structural rules)

일반 에디터의 들여쓰기는 사용자가 스페이스/탭을 친 결과다. 원고지에서는 **줄의 위치(몇 째 줄인지)와
문맥에 따라 비워야 할 칸이 규칙으로 정해져 있다.** 이미지에 나온 규칙:

### 머리말(글머리) 영역 — 줄 번호별 고정 약속
| 줄 | 내용 | 규칙 |
|----|------|------|
| 1째 줄 | 글 종류 (예: 독후감) | **첫 칸 한 칸 띄우고** 적음 |
| 2째 줄 | 제목 | **가운데 정렬**, 마침표 안 찍음 |
| 3째 줄 | 학교 | **오른쪽 끝에서 2~3칸 남기고** 적음(우측 정렬) |
| 4째 줄 | 학년·반·번호 | 오른쪽 끝 2~3칸 남기고 적음 |
| 5째 줄 | 이름 | 오른쪽 끝 2~3칸 남기고 적음 |
| (본문 전) | — | **한 줄을 통째로 비움** |

### 본문 규칙
- 본문 **처음 시작** 시 첫 칸을 비운다(1칸 들여쓰기).
- **새 문단**이 시작될 때마다 첫 칸을 비운다.
- **대화문(따옴표)이 시작될 때**도 첫 칸을 비운다.

### 일반 에디팅과의 차이
- 일반 에디터의 들여쓰기는 "자유로운 공백"이지만, 원고지는 **줄 번호에 의미가 부여**된다.
  → 머리말 영역은 사실상 **양식(template)** 으로 다루는 게 자연스럽다(자동 정렬/자동 빈 칸).
- 문단 들여쓰기가 "스페이스 1개"가 아니라 **"첫 칸 1개를 비운다"는 칸 단위 약속**이다.
  엔터(문단 바꿈) 시 자동으로 다음 줄 첫 칸을 비우는 동작이 필요하다.

---

## 3. 행두·행말 금칙 (禁則, kinsoku) — 줄 경계 처리

일반 에디터의 자동 줄바꿈은 "폭이 넘치면 다음 줄로" 단순하다. 원고지는 **줄의 첫 칸/마지막 칸에
올 수 없는 글자**가 있어서 줄바꿈 위치가 보정된다.

이미지에 명시된 것:
- **마지막 칸에서 문장이 끝나면**, 마침표를 **같은 칸에 글자와 함께** 쓰거나 **바깥 여백**에 쓴다.
  → 즉 마침표·쉼표·닫는 따옴표는 **줄 맨 앞으로 밀려나면 안 된다**(행두 금칙).
- 마침표·쉼표는 한 칸을 쓰되 **다음 칸을 띄우지 않는다** → 줄 끝 공백 처리가 일반 텍스트와 다르다.

### 일반 에디팅과의 차이
- 일반 줄바꿈: 넘치는 글자를 그냥 다음 줄로 보냄.
- 원고지: 다음 줄 첫 칸에 구두점이 오게 되면, 그 구두점을 **앞 줄 마지막 칸에 끼워 넣거나 여백 칸**으로
  처리해야 한다. → 줄바꿈 계산에 **"이 글자가 줄 첫 칸에 와도 되는가"** 판정이 추가된다.

## 4. 커서·선택·삭제 모델

| 항목 | 일반 텍스트 에디터 | 원고지 |
|------|-------------------|--------|
| 커서 위치 | 문자 오프셋(문자 사이) | **칸 인덱스 (+ 칸 내부 좌/우 위치)** |
| 한 칸 = 한 글자? | 개념 없음 | 아님 — 숫자/소문자는 칸 내부 2위치 |
| 줄바꿈 | `\n` 문자 1개 | **문단 바꿈** = 새 줄 + 첫 칸 들여쓰기(구조 동작) |
| 빈 칸 클릭 | 해당 없음 | 내용 범위 밖 칸 클릭 시 글 끝으로 캐럿 이동(현재 구현됨) |
| 정렬 | 좌→우 흐름 | 칸 안에서 **가운데 정렬**(2글자면 좌우 배치) |

### 현재 구현(`GridEditor.vue`)과의 간극
현재는 `layoutText()`가 **1글자 = 1칸**, `\n` = 줄바꿈만 처리한다. 위 규칙을 지원하려면:

1. **토큰 분류기**: 입력 문자열을 (한글 | 숫자 | 대문자 | 소문자 | 구두점 | 따옴표 | 줄임표 | 공백 | 줄바꿈)으로 나눈다.
2. **칸 배치기(layout)**: 토큰을 점유 규칙대로 칸에 채운다(숫자/소문자 2개 묶기, 마침표+따옴표 합치기 등).
3. **금칙 보정**: 줄바꿈 직전에 행두/행말 금칙을 검사해 위치를 조정한다.
4. **머리말 양식(선택)**: 글 종류/제목/학교/학년·반/이름 줄을 자동 정렬·자동 빈 칸 처리.
5. **칸 내부 위치를 포함한 캐럿 모델**: `offset ↔ {cell, subIndex}` 양방향 매핑.

이 중 1~3이 "실제 원고지처럼 타이핑"의 핵심이며, 4는 양식 기능, 5는 정밀 편집 단계에서 필요하다.

## 5. 구현 우선순위 제안

| 단계 | 범위 | 난이도 |
|------|------|--------|
| A | 칸 점유 규칙(숫자 2·소문자 2·구두점) — `layoutText` 확장 | 중 |
| B | 문단 들여쓰기 자동화(엔터 시 첫 칸 비움) | 하 |
| C | 행두 금칙(마침표·닫는 따옴표가 줄 첫 칸에 안 오게) | 중 |
| D | 칸 내부 2글자 캐럿/삭제 정밀화 | 상 |
| E | 머리말 양식(제목 가운데·이름 우측 정렬) | 중 (별도 모드) |

> 참고: macOS 한글 IME 조합 관련 제약은 [korean-ime-bug.md](korean-ime-bug.md) 참고.
> 칸 점유 규칙이 들어가면 "조합 중 글자가 어느 칸을 차지하는가"도 토큰 분류기를 거쳐야 하므로
> IME 조합 셀 계산(`composingText` → 활성 셀)도 함께 손봐야 한다.
